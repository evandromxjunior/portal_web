ENDPOINTS DO SISTEMA - CIA DO SILK
==================================

1. PORTAL WEB
-------------

Interface web usada pelo cliente para consultar boletos:

GET http://localhost:5173/


2. API LOCAL DO SISTEMA
-----------------------

2.1. Verificar se a API esta online

GET http://localhost:3333/api/health

Exemplo de resposta:

{
  "status": "ok",
  "mode": "winthor-oracle"
}


2.2. Consultar boletos em aberto por CPF/CNPJ

GET http://localhost:3333/api/receivables?document=CPF_OU_CNPJ

Exemplo:

GET http://localhost:3333/api/receivables?document=85862589546

Retorna:
- Cliente
- Documento
- Titulos em aberto
- Vencimento
- Valor
- Linha digitavel
- URL do PDF, quando localizado


2.3. Abrir PDF do boleto

GET http://localhost:3333/api/receivables/:id/pdf

Exemplo:

GET http://localhost:3333/api/receivables/13341-633757-Z/pdf

Observacao:
Esse endpoint so funciona quando o PDF do boleto for encontrado na pasta configurada:

\\10.0.1.3\winthor-share-file\Winthor


2.4. Webhook reservado para WhatsApp

POST http://localhost:3333/api/whatsapp/webhook

Observacao:
Rota reservada para futura integracao com WhatsApp.


2.5. Consulta de cliente usando API TOTVS/WTA

GET http://localhost:3333/api/winthor-api/customers?document=CPF_OU_CNPJ

Exemplo:

GET http://localhost:3333/api/winthor-api/customers?document=85862589546


3. API TOTVS/WTA
----------------

3.1. Login para gerar token

POST http://IP_DO_WTA:PORTA/winthor/autenticacao/v1/login

Body:

{
  "login": "USUARIO",
  "senha": "SENHA_MD5"
}

Resposta esperada:

{
  "accessToken": "chaveToken"
}

Configuracao no .env:

TOTVS_API_BASE_URL=http://IP_DO_WTA:PORTA
TOTVS_API_LOGIN_PATH=/winthor/autenticacao/v1/login
TOTVS_API_LOGIN_USER=usuario
TOTVS_API_LOGIN_PASSWORD=senha_md5
TOTVS_API_PASSWORD_MODE=already_md5
TOTVS_API_LOGIN_USER_FIELD=login
TOTVS_API_LOGIN_PASSWORD_FIELD=senha
TOTVS_API_TOKEN_FIELD=accessToken


3.2. Consulta de cliente na API oficial TOTVS

GET http://IP_DO_WTA:PORTA/api/wholesale/v1/customer/list?personIdentificationNumber=CPF_OU_CNPJ

Exemplo:

GET http://IP_DO_WTA:PORTA/api/wholesale/v1/customer/list?personIdentificationNumber=85862589546


4. CONEXAO ORACLE WINTHOR
-------------------------

Nao e endpoint HTTP. E a string de conexao usada pelo backend para consultar o banco Oracle do WINTHOR.

ORACLE_CONNECT_STRING=201.157.219.14:1521/CG6V4Y_195977_W_high.paas.oracle.com

Tabelas principais consultadas:

- PCCLIENT
- PCPREST

Campos importantes encontrados:

- PCCLIENT.CGCENT
- PCPREST.CODBARRA
- PCPREST.CODBARRA2
- PCPREST.BOLETO
- PCPREST.NOSSONUMBCO
- PCPREST.NOSSONUMBCO2
- PCPREST.PASTAARQUIVOBOLETO


5. COMANDOS UTEIS
-----------------

Subir portal e API:

npm run dev

Gerar token TOTVS/WTA:

npm run totvs:token

Diagnosticar campos de boleto no WINTHOR:

npm run winthor:diagnose-boleto

Consultar documento no WINTHOR:

npm run winthor:consulta-documento -- 85862589546

Inspecionar dados de arquivo/PDF de boleto:

npm run winthor:inspect-boleto -- 633757


6. OBSERVACOES
--------------

- O portal web usa a API local em http://localhost:3333.
- A API local consulta o Oracle do WINTHOR quando USE_MOCK_DATA=false.
- O botao "Copiar linha" funciona quando PCPREST.CODBARRA retorna valor.
- O botao "Abrir PDF" depende de localizar o arquivo PDF na pasta compartilhada configurada.
- A integracao WhatsApp ainda esta reservada para uma etapa futura.
